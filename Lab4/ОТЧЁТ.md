## Характеристики системы тестирования
- **Платформа**: Windows/Linux/MacOS
- **Python**: 3.8+
- **Процессор**: Intel Core i5/i7 или аналогичный
- **Память**: 8GB+ RAM

## Реализованные алгоритмы

### 1. Сортировка пузырьком (Bubble Sort)
**Сложность**: O(n²) во всех случаях, O(n) в лучшем (отсортированный массив)  
**Память**: O(1)  
**Устойчивость**: Да  
**Особенности**: Простая реализация, неэффективна на больших данных

### 2. Сортировка выбором (Selection Sort)
**Сложность**: O(n²) во всех случаях  
**Память**: O(1)  
**Устойчивость**: Нет  
**Особенности**: Всегда делает O(n) обменов

### 3. Сортировка вставками (Insertion Sort)
**Сложность**: O(n²) в худшем, O(n) в лучшем  
**Память**: O(1)  
**Устойчивость**: Да  
**Особенности**: Эффективна на маленьких и почти отсортированных массивах

### 4. Сортировка слиянием (Merge Sort)
**Сложность**: O(n log n) во всех случаях  
**Память**: O(n)  
**Устойчивость**: Да  
**Особенности**: Стабильная производительность, требует доп. памяти

### 5. Быстрая сортировка (Quick Sort)
**Сложность**: O(n log n) в среднем, O(n²) в худшем  
**Память**: O(log n)  
**Устойчивость**: Нет  
**Особенности**: Быстрее всех в среднем случае, неустойчив

Ответы на контрольные вопросы
1. Какие алгоритмы сортировки имеют сложность O(n²) в худшем случае, а какие - O(n log n)?
O(n²): Bubble Sort, Selection Sort, Insertion Sort (в худшем случае). O(n log n): Merge Sort (всегда), Quick Sort (в среднем случае), Heap Sort. Merge Sort гарантирует O(n log n), тогда как Quick Sort может деградировать до O(n²) при неудачном выборе опорного элемента.

2. Почему сортировка вставками эффективна для маленьких или почти отсортированных массивов?
Insertion Sort эффективна потому что в лучшем случае (уже отсортированный массив) имеет сложность O(n) - каждый элемент только сравнивается и вставляется на свое место. Для почти отсортированных данных количество операций сдвига минимально. На маленьких массивах константные факторы преобладают над асимптотической сложностью.

3. В чем разница между устойчивой и неустойчивой сортировкой?
Устойчивая сортировка сохраняет относительный порядок элементов с одинаковыми ключами. Например, если мы сортируем студентов по оценкам, и двое имеют одинаковую оценку, устойчивая сортировка сохранит их исходный порядок. Устойчивые: Bubble Sort, Insertion Sort, Merge Sort. Неустойчивые: Selection Sort, Quick Sort.

4. Опишите принцип работы Quick Sort. Что такое "опорный элемент"?
Quick Sort использует стратегию "разделяй и властвуй": выбирается опорный элемент (pivot), массив разбивается на две части - элементы меньше опора и больше опора, затем рекурсивно сортируются обе части. Опорный элемент - это элемент, относительно которого происходит разделение. Его выбор критически важен: хороший выбор (медиана) дает O(n log n), плохой (минимум/максимум) ведет к O(n²).

5. Когда Merge Sort предпочтительнее Quick Sort?
Merge Sort предпочтительнее когда: 1) Требуется гарантированное O(n log n) время; 2) Данные могут быть частично отсортированы (Quick Sort деградирует); 3) Нужна устойчивая сортировка; 4) Работаем с внешней памятью (внешняя сортировка); 5) Доступна дополнительная память O(n).
